一、设计

库存预热：
提前把商品添加到redis中，整个秒杀流程都在redis中操作，秒杀结束，在异步去修改库存。

用户下单：
为了提升下单速度，将订单存放的redis中，用户支付后，再将订单存到mysql中，并清空redis中的订单。

创建订单：
用户每次下单的时候，使用rabbitMQ创建一个队列，采用异步的方式创建订单，从rabbitMQ中读取信息，创建订单。

下单状态：
做一个页面的判断，每个1s查询下单状态。

超卖问题：
数据库添加唯一索引，减库存的时候先进行库存数量判断；用redis来维护库存，redis中的库存不足则秒杀失败；异步队列处理订单。

二、缓存

在高并发的情况下必然会遇到缓存雪崩、缓存击穿、缓存穿透等问题。而且秒杀是读多写少，使用redis作为缓存非常合适（为了避免redis服务器出问题，可以使用redis集群）。

2.1、缓存穿透

描述：
是指缓存和数据库中没有数据，而用户不断请求，会导致数据库压力过大
解决方案：  
1、在接口层校验，如果用户鉴权校验，id为基础校验，id<=0的直接拦截
2、从缓存取不到数据，在数据库中也没有取到，可以将key-value对写成key-null，将缓存有效时间可以设置短点，设置太长会导致无法正常使用。

2.2、缓存击穿

描述：
是指缓存中没有数据但是数据库中有数据（一般是缓存时间到期），这是由于并发用户特别多，同时读缓存没有读到数据，有同时去请求数据库，引起数据库压力瞬间增大。
解决方案：
1、设置热点永不过期
2、加互斥锁

    public String getData(String key) throws InterruptedException {
        // 创建锁
        ReentrantLock reenLock = new ReentrantLock();
        // 从缓存中读取数据
        String result = getDataFromRedis(key);
        // 缓存中不存在数据
        if (result == null){
            // 获取锁，获取成功去数据库中取数据
            if (reenLock.tryLock()){
                // 从数据库获取数据
                result = getDataFromMysql(key);
                // 更新缓存
                if (result != null){
                    setDataToCache(key,result);
                }
                // 释放锁
                reenLock.unlock();
            }else{
                // 暂停一会，在重新去获取
                Thread.sleep(100);
                result = getData(key);
            }
        }
        return result;
    }

2.3、缓存雪崩

描述：
是指缓存中数据大量到过期时间，而查询数据量巨大，引起数据库压力过大甚至宕机  
解决方案：  
1、缓存数据的过期时间设置随机，防止同一时间大量数据过期。  
2、缓存数据库是分布式部署，将热点数据均匀分布不同缓存数据库中  
3、设置热点数据永不过期

三、削峰/限流

秒杀流量在某一时刻非常高，让瞬间进来的流量进到一个缓存池，然后进行平缓处理（mq消息队列处理）。

@Configuration
public class RabbitConfig {

    private static final Logger logger = LoggerFactory.getLogger(RabbitConfig.class);
    // 交换机
    public final static String EXCHANGE_NAME = "order:mq:exchange";
    // 队列
    public final static String QUEUE_NAME = "order:mq:queue";
    // 路由键
    public final static String ROUTING_KEY = "order:mq:routing:key";



    /**
     * channel链接工厂
     */
    @Autowired
    private CachingConnectionFactory connectionFactory;

    /**
     * 监听器容器配置
     */
    @Autowired
    private SimpleRabbitListenerContainerFactoryConfigurer factoryConfigurer;


    /**
     * 声明rabbittemplate
     * @return
     */
    @Bean
    public RabbitTemplate rabbitTemplate(){
        //消息发送成功确认，对应application.properties中的spring.rabbitmq.publisher-confirms=true
        connectionFactory.setPublisherConfirms(true);
        //消息发送失败确认，对应application.properties中的spring.rabbitmq.publisher-returns=true
        connectionFactory.setPublisherReturns(true);
        RabbitTemplate rabbitTemplate = new RabbitTemplate(connectionFactory);
        //设置消息发送格式为json
        rabbitTemplate.setMessageConverter(new Jackson2JsonMessageConverter());
        rabbitTemplate.setMandatory(true);
        //消息发送到exchange回调 需设置：spring.rabbitmq.publisher-confirms=true
        rabbitTemplate.setConfirmCallback(new RabbitTemplate.ConfirmCallback() {
            @Override
            public void confirm(CorrelationData correlationData, boolean ack, String cause) {
                logger.info("消息发送成功:correlationData({}),ack({}),cause({})",correlationData,ack,cause);
            }
        });
        //消息从exchange发送到queue失败回调  需设置：spring.rabbitmq.publisher-returns=true
        rabbitTemplate.setReturnCallback(new RabbitTemplate.ReturnCallback() {
            @Override
            public void returnedMessage(Message message, int replyCode, String replyText, String exchange, String routingKey) {
                logger.info("消息丢失:exchange({}),route({}),replyCode({}),replyText({}),message:{}",exchange,routingKey,replyCode,replyText,message);
            }
        });
        return rabbitTemplate;
    }

    //---------------------------------------订单队列------------------------------------------------------

    /**
     * 声明订单队列的交换机
     * @return
     */
    @Bean
    public TopicExchange orderTopicExchange(){
        //设置为持久化 不自动删除
        return new TopicExchange(EXCHANGE_NAME,true,false);
    }

    /**
     * 声明订单队列
     * @return
     */
    @Bean
    public Queue orderQueue(){
        return new Queue(QUEUE_NAME,true);
    }

    /**
     * 将队列绑定到交换机
     * @return
     */
    @Bean
    public Binding simpleBinding(){
        return BindingBuilder.bind(orderQueue()).to(orderTopicExchange()).with(ROUTING_KEY);
    }
}

    // 生产消息
    public String send(String message){
        rabbitTemplate.convertAndSend(RabbitConfig.EXCHANGE_NAME,RabbitConfig.ROUTING_KEY,message);
        retuen "消息发送成功";
    }
    
    

    // 配置rabbitmq监听
    /**
     * 处理接收到的消息
     * @param message 消息体
     * @param channel 通道，确认消费用
     */
    @RabbitListener(queues = RabbitConfig.QUEUE_NAME)
    public void onMessage(Message message, com.rabbitmq.client.Channel channel){
        try{
            //获取交付tag
            long tag = message.getMessageProperties().getDeliveryTag();
            String str = new String(message.getBody(),"utf-8");
            logger.info("接收到的消息：{}",str);
            //确认消费
            channel.basicAck(tag,true);
        }catch(Exception e){
            logger.error("消息监听确认机制发生异常：",e.fillInStackTrace());
        }
    }

    /**
       * 容器工厂配置 并发
       * @return
    */
    @Bean("containerFactory")
    public RabbitListenerContainerFactory rabbitListenerContainerFactory(ConnectionFactory connectionFactory){
        SimpleRabbitListenerContainerFactory listenerContainerFactory = new SimpleRabbitListenerContainerFactory();
        listenerContainerFactory.setConnectionFactory(connectionFactory);
        listenerContainerFactory.setMessageConverter(new Jackson2JsonMessageConverter());
        // 最大并发数
        listenerContainerFactory.setMaxConcurrentConsumers(20);
        // 初始并发数
        listenerContainerFactory.setConcurrentConsumers(10);
        return listenerContainerFactory;
    }
