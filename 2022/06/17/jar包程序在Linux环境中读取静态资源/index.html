jar包程序在Linux环境中读取静态资源

Linux jar包 静态资源
问题记录
发布日期:   2022-06-17
文章字数:   363
阅读时长:   1 分
阅读次数:  

问题描述
将xlsx文件放到了resource目录下，在本地是可以读取到的，打成jar包之后访问不到

解决办法

1.在pom.xml文件中导入相关坐标和插件，保证不进行替换静态文件的属性，将资源打包进入jar包中

<!--配置打包成jar包之后读取文件-->
<resources>
    <resource>
        <directory>src/main/java</directory>
        <includes>
            <include>**/*.properties</include>
            <include>**/*.xml</include>
            <include>**/*.yml</include>
        </includes>
        <!--不替换资源中的属性-->
        <filtering>false</filtering>
    </resource>
    <resource>
        <directory>src/main/resources</directory>
        <includes>
            <include>**/*.*</include>
        </includes>
        <!--不替换资源中的属性-->
        <filtering>false</filtering>
    </resource>
</resources>

2.修改读取静态资源的方式

  • 使用ClassPathResource进行静态文件的读取，因为这个类的对象是用于获取class字节码文件路径的文件
// resource下的文件路径
ClassPathResource classPathResource = new ClassPathResource("static/123.xlsx");
try {
  InputStream inputStream = classPathResource.getInputStream();
  MultipartFileUtil.inputStreamToFile(inputStream, new File("./焊接工艺数据库模板.xlsx"));
} catch (IOException e) {
  e.printStackTrace();
}
  • 其中MultipartFileUtil.inputStreamToFile(),此方法是封装的工具类.工具类代码如下:
      public static void inputStreamToFile(InputStream ins, File file) {
      OutputStream os = null;
      try {
          os = new FileOutputStream(file);
          int bytesRead = 0;
          byte[] buffer = new byte[1024];
          while ((bytesRead = ins.read(buffer, 0, 1024)) != -1) {
              os.write(buffer, 0, bytesRead);
          }
          os.close();
          ins.close();
      } catch (FileNotFoundException e) {
          e.printStackTrace();
      } catch (IOException e) {
          e.printStackTrace();
      }
  }
